5. CORE PIPELINE (Production - FastAPI Backend)
================================================================================

   The system follows a 5-step pipeline via POST /voice-query:

   Step 1: Speech-to-Text (Deepgram)
           -> Audio bytes sent to Deepgram Nova-2 API
           -> Automatic language detection (English vs Urdu)
           -> Returns transcript + detected language code
           -> File: backend/stt.py

   Step 2: Language Handling
           -> If English: transcript used as-is for display and RAG
           -> If Urdu: Arabic script converted to Roman Urdu via GPT-4o-mini
           -> Roman Urdu used for both display (transcript) and RAG query
           -> File: backend/transliterate.py

   Step 3: Retrieval-Augmented Generation (RAG)
           -> Query embedded and searched against ChromaDB (top-15 results)
           -> GPT-4o-mini generates response using retrieved context ONLY
           -> If Urdu: returns both roman_urdu (display) and arabic_urdu (TTS)
           -> If English: returns English text for both display and TTS
           -> Anti-hallucination: "Mujhe is bare mein kafi maloomat nahi hai"
           -> File: backend/rag.py

   Step 4: Text-to-Speech (ElevenLabs)
           -> Response text sent to ElevenLabs multilingual v2 model
           -> Voice: Sarah (EXAVITQu4vr4xnSDxMaL) - female, mature, reassuring
           -> Supports both English and Urdu pronunciation
           -> Audio returned as base64 data URL (no static file serving needed)
           -> File: backend/tts.py

   Step 5: Response Delivery
           -> JSON returned: { transcript, response_text, audio_url }
           -> audio_url is a base64-encoded data:audio/mpeg URL
           -> Frontend auto-plays the audio and displays text
           -> File: backend/main.py

   API Endpoint:
     POST http://localhost:8000/voice-query
     Input:  multipart/form-data with "audio" file field
     Output: { "transcript": "...", "response_text": "...", "audio_url": "data:audio/mpeg;base64,..." }

   Health Check:
     GET http://localhost:8000/health
     Output: { "status": "ok" }

8. BACKEND IN DETAIL (backend/ package)
================================================================================

   A. backend/config.py
   --------------------
   - Loads .env from project root via python-dotenv
   - Exports all API keys: OPENAI, DEEPGRAM, ELEVENLABS
   - ChromaDB config: persist_directory, collection "sample"
   - LLM config: GPT-4o-mini, RAG_TOP_K=15
   - ElevenLabs config: Sarah voice (EXAVITQu4vr4xnSDxMaL), multilingual_v2

   B. backend/stt.py
   -----------------
   - transcribe_audio(audio_bytes, mimetype) -> {"transcript", "language"}
   - Uses httpx POST to Deepgram REST API
   - Params: model=nova-2, smart_format=true, detect_language=true
   - Returns "ur" if Urdu/Hindi detected, "en" otherwise

   C. backend/transliterate.py
   ---------------------------
   - urdu_to_roman(urdu_text) -> str : Arabic script to Roman Urdu
   - roman_to_urdu(roman_text) -> str : Roman Urdu to Arabic script (for TTS)
   - Both use GPT-4o-mini with temperature=0.0

   D. backend/rag.py
   -----------------
   - ChromaDB vector store initialized once at module load
   - generate_response(query, language) -> {"response_text", "tts_text"}
   - Urdu path: GPT generates JSON with roman_urdu + arabic_urdu keys
   - English path: GPT generates plain English text
   - response_text = display text, tts_text = text for speech synthesis

   E. backend/tts.py
   -----------------
   - synthesize_speech(text) -> str (base64 data URL)
   - Uses httpx POST to ElevenLabs REST API
   - Voice: Sarah (premade, female, mature, reassuring)
   - Model: eleven_multilingual_v2 (English + Urdu)
   - Returns "data:audio/mpeg;base64,..." string

   F. backend/main.py
   ------------------
   - FastAPI app with CORS (localhost:5173)
   - POST /voice-query endpoint orchestrates full pipeline
   - Detailed step-by-step logging to stdout
   - Error handling: 400 (no speech), 502 (external API), 500 (internal)
   - GET /health endpoint for health checks
   - Response model: Pydantic VoiceQueryResponse

   Run command:
     python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload

13. LANGUAGE BEHAVIOR
================================================================================

    The system dynamically adapts based on detected input language:

    IF user speaks ENGLISH:
      - Deepgram detects "en"
      - Transcript displayed in English
      - RAG response generated in English
      - TTS audio synthesized in English

    IF user speaks URDU:
      - Deepgram detects "ur" (or "hi" as fallback)
      - Urdu script converted to Roman Urdu via GPT-4o-mini
      - Transcript displayed in Roman Urdu
      - RAG response generated in Roman Urdu
      - Roman Urdu converted to Arabic Urdu script for TTS
      - TTS audio synthesized in natural Urdu

    Anti-hallucination:
      - If info not in knowledge base: "Mujhe is bare mein kafi maloomat nahi hai"
      - Verified working with test inputs #6, #7, #8 (out-of-KB queries)


14. KNOWN ISSUES & NOTES
================================================================================

    1. LANGCHAIN DEPRECATION WARNING (cosmetic):
       backend/rag.py uses langchain_community.vectorstores.Chroma which
       shows a deprecation warning. Non-blocking; migrate to langchain-chroma
       package in the future.

    2. TYPESCRIPT CONFIG BUT JSX FILES:
       tsconfig.json configured but all source files use .jsx. TypeScript
       strict mode is disabled.

    3. REMOVED DOCUMENTATION:
       Several markdown files referenced in QUICK_START.md were removed in
       commit e094a79 (FYP_TEST_CASES.md, etc.).

    4. ELEVENLABS FREE TIER:
       The original "Rachel" voice (21m00Tcm4TlvDq8ikWAM) required a paid
       plan. Switched to "Sarah" (EXAVITQu4vr4xnSDxMaL), a premade voice
       available on free accounts.

    5. BASE64 AUDIO DELIVERY:
       TTS audio is returned as base64 data URLs in JSON. This avoids
       static file serving but adds ~33% size overhead. Typical responses
       are 50-200KB MP3 (~67-267KB as base64), which is acceptable.

    6. CORS CONFIGURED FOR DEV ONLY:
       Backend allows origins localhost:5173 and 127.0.0.1:5173. For
       production, this should be updated to the actual domain.
