5. CORE PIPELINE (How It Works)
================================================================================

   The system follows an 8-step pipeline:

   Step 1: User Speaks
           -> Voice input captured via browser Web Audio API

   Step 2: Audio Packaging
           -> Audio compressed into .ogg format and sent to backend

   Step 3: Speech-to-Text (STT)
           -> OpenAI Whisper API transcribes Urdu audio into Urdu script
           -> Function: transcribe_audio() in pipeline.ipynb

   Step 4: Urdu Script to Roman Urdu
           -> GPT-4o-mini converts Arabic Urdu script to Roman Urdu
           -> Function: urdu_to_roman() in pipeline.ipynb

   Step 5: Retrieval-Augmented Generation (RAG)
           -> Roman Urdu query is embedded and searched against ChromaDB
           -> Top 15 similar documents retrieved
           -> GPT-4o-mini generates a bilingual response (Roman Urdu + Arabic Urdu)
              using ONLY the retrieved context (no hallucination)
           -> Function: generate_response() in pipeline.ipynb

   Step 6: Response Formatting
           -> Response returned as JSON with "roman_urdu" and "arabic_urdu" keys
           -> If information is not in context, responds:
              "Mujhe is bare mein kafi maloomat nahi hai"

   Step 7: Text-to-Speech (TTS)
           -> Arabic Urdu response synthesized into .mp3 audio using gTTS
           -> Function: synthesize_urdu_to_mp3() in pipeline.ipynb

   Step 8: Audio Playback
           -> Synthesized audio played back to the user in the browser


6. KNOWLEDGE BASE (script.txt)
================================================================================

   The RAG system's knowledge base is a single text file (script.txt, ~50KB,
   46 lines) written entirely in Roman Urdu. It covers:

   - Attendance policies (80% UG, 75% PG)
   - Academic calendar and semester structure
   - Grading system (A+ to F, GPA scales)
   - Honors and Dean's/Rector's lists
   - Academic warning and dismissal rules
   - Registration, course add/drop/withdrawal policies
   - Final Year Project registration rules
   - Summer semester policies
   - Admission process and merit criteria
   - Admission tests (NU-OAT) structure and subject breakdown
   - Merit list calculation weights
   - Fee structure (Rs. 11,000/credit hour, admission fee Rs. 30,000, etc.)
   - Fee refund policy and schedule
   - Scholarships (internal and external, 15+ schemes listed)
   - Study loan scheme (Qarz-E-Hasna)
   - Dress code policy
   - Degree programs offered (BS, MS, PhD in Computing disciplines)
   - MS program requirements (30-60 credit hours, thesis/project)
   - PhD program requirements (18 credit hours coursework, comprehensive exam)
   - Generative AI usage policies
   - Academic dishonesty and disciplinary actions
   - Transfer and readmission policies
   - 2025 admission schedule (May 19 - August 18)

   This text is semantically chunked in main.ipynb using GPT-4o-mini into
   247 chunks, each with a category, content, and summary. These chunks are
   embedded using OpenAI Embeddings and stored in ChromaDB under the
   collection name "sample".

8. NOTEBOOKS IN DETAIL
================================================================================

   A. main.ipynb (388 KB)
   ----------------------
   Purpose: Builds the RAG pipeline from scratch.

   Cell-by-cell flow:
   1. Import LangChain, OpenAI, dotenv; load environment
   2. Load script.txt with robust encoding fallback
   3. Split document into batches by newline
   4. Define JSON schema for semantic chunking (category, content, summary)
   5. Process each batch through GPT-4o-mini for semantic chunking
   6. Combine all chunks (247 total)
   7. Convert chunks to LangChain Document objects with metadata
   8. Initialize ChromaDB vector store
   9. Add documents to vector store
   10. Test similarity search with query "fast kee tuition fee kitne hain?"
   11. Full RAG pipeline: retrieve -> prompt -> structured output
       (bilingual: roman_urdu + arabic_urdu)

   Output example:
     Query: "fast kee tuition fee kitne hain?"
     Answer: "FAST ki tuition fee Rs. 11,000 per credit hour hai."

   B. pipeline.ipynb (1.17 MB)
   ---------------------------
   Purpose: Complete end-to-end STT -> RAG -> TTS pipeline.

   Functions defined:
   1. transcribe_audio()    - Whisper STT (Urdu audio -> Urdu script)
   2. urdu_to_roman()       - GPT-4o-mini (Urdu script -> Roman Urdu)
   3. generate_response()   - ChromaDB retrieval + GPT-4o-mini generation
   4. synthesize_urdu_to_mp3() - gTTS (Arabic Urdu text -> MP3)

   Single query test:
     Input: tuition_fee.ogg
     Transcribed: "Fast University ki tuition fees kitni hai?"
     Response: "FAST University ki tuition fees Rs. 11000 per credit hour hai."
     Output: response_urdu.mp3

   Batch processing:
     Processes all 8 files in Test_cases_inputs/
     Generates corresponding response MP3s in test_results/
     Saves summary to test_results/summary.json

    formula.txt
    -----------
    Contains the formula for calculating total parameters of GPT-2 Small (124M):
      Total Params = (vocab_size x emb_dim) + (context_length x emb_dim) +
                     n_layers x (4 x emb_dim^2 + 2 x (emb_dim x 4*emb_dim)) +
                     2 x emb_dim
    Where: vocab_size=50257, context_length=1024, emb_dim=768, n_layers=12
