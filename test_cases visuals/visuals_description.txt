              that the system meets its quality bar before the
              examiner reads any detail.

 07 — 07_unit_integration_duration.png
----------------------------------------------------------------
Title       : Unit & Integration - Execution Time per Case
Type        : Vertical bar chart on a log y-axis
Data source : data.unit_integration.cases[].result.duration_ms
What it shows :
    Per-case execution time for all 12 unit-and-integration
    cases. The log scale is essential because pure-Python unit
    tests (UT-02 to UT-06) execute in 0-1 ms while integration
    tests that hit the LLM endpoint (IT-01) take ~9 seconds.
    Each bar is colour-coded by outcome: green = pass,
    red = fail, yellow = skip.

Observable structure:
    - UT-01 ~ 1.5 s   (transliteration warm-up; first call)
    - UT-02..UT-06 ~ 0-1 ms (pure-Python helpers)
    - IT-01 ~ 9 s    (real /text-query call to the LLM)
    - IT-02, IT-06   skipped - audio fixtures missing
    - IT-03..IT-05   ~ 6-13 ms (validation paths, no LLM)

Caption     : Fig X.7 - Execution time per case for the unit and
              integration suite (logarithmic y-axis). The gap of
              four orders of magnitude between pure-Python
              helpers and full-stack integration calls justifies
              isolating expensive cases behind the --quick flag.

Interpret   : Visually proves why --quick exists: 99 % of the
              suite's run time is in a single integration call.

 09 — 09_answer_accuracy_keywords.png
----------------------------------------------------------------
Title       : Answer Accuracy - Matched vs Expected Keywords
              (red bars = case failed)
Type        : Grouped bar chart, one pair per case
Data source : data.answer_accuracy.cases[].result.matched_keywords
              vs case.expected_contains
What it shows :
    Per-case keyword recall. The grey bar is the number of gold
    keywords the case expects to find in the response; the
    coloured bar is how many were actually present. A case
    passes when matched >= ceil((expected + 1) / 2), which is
    visually clear here because failed cases have a noticeably
    shorter coloured bar (and are filled red).

Failures in the current run: AN-01, AN-10.

Caption     : Fig X.9 - Keyword recall per answer-accuracy case.
              Two of the twelve cases fall below the 50 % match
              threshold: AN-01 ('SSC', 'admission test') and
              AN-10 ('4', 'years'). Both are scoring artefacts
              rather than semantic errors - the model produces
              correct Roman-Urdu paraphrases that miss the exact
              token form.

Interpret   : Sets up the discussion in the limitations section
              about moving from lexical to semantic scoring
              (e.g., cosine similarity over embeddings).
