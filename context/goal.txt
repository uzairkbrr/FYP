PROJECT GOAL: MAHIR ON CALL (FYP)

Objective:
Build a production-like voice-based AI assistant for FAST-NUCES Peshawar that can handle student queries using voice input and output in Urdu and English.

--------------------------------------------------

1. USER INTERACTION FLOW

The system must support TWO interaction entry points:

A. Voice Section (existing UI)
- User can interact using the existing "voice-section" in the main UI

B. Chat Widget (floating)
- User can interact using the floating chat widget

For BOTH interaction methods:

- User clicks microphone icon
- User records a voice note (maximum 30 seconds)
- Recording stops manually or automatically at 30 seconds
- Audio is sent to backend
- System processes the query
- System returns:
  - Transcript
  - AI-generated response
  - Audio response (TTS)
- Audio is played automatically
- Chat-style UI displays conversation (user + assistant)

--------------------------------------------------

2. LANGUAGE BEHAVIOR (STRICT RULE)

The system must dynamically adapt based on user language:

IF user speaks in ENGLISH:
- Transcript must be in English
- Response must be in English
- Audio response must be in English

IF user speaks in URDU:
- Transcript must be in Roman Urdu
- Response must be in Roman Urdu
- Audio response must be in Urdu (natural spoken form)

--------------------------------------------------

3. FRONTEND REQUIREMENTS

A. EXISTING UI (DO NOT MODIFY)
- Keep current React UI exactly the same
- Do NOT refactor or restructure existing components

B. VOICE SECTION (IMPORTANT)
- Existing voice-section must be connected to backend
- Must support full voice interaction (same as chat widget)

C. FLOATING CHAT WIDGET
- Add a floating AI icon at bottom-right corner
- Clicking icon opens a chat widget overlay
- Widget behaves like Messenger/Intercom style

D. CHAT WIDGET FEATURES
- Microphone button (start/stop recording)
- Display recording timer (0–30 seconds)
- Show "Recording..." state
- Show "Processing..." state
- Display transcript
- Display AI response text
- Play response audio
- Show error if microphone permission denied

E. CONTACT MODERATOR FEATURE (NEW)
- Add a button: "Contact Moderator"

On click:
1. Email option:
   - Open user's email client
   - Send email to: admissions@pwr.nu.edu.pk

2. Call option:
   - Provide click-to-call functionality
   - Phone number: (091) 111 128 128

--------------------------------------------------

4. BACKEND REQUIREMENTS

Convert notebook-based pipeline into a production API using FastAPI.

Single endpoint required:

POST /voice-query

Input:
- Audio file (.ogg or .wav)

Output (JSON):
{
  "transcript": "...",
  "response_text": "...",
  "audio_url": "..."
}

--------------------------------------------------

5. PIPELINE (STRICT ORDER)

Step 1: Speech-to-Text (STT)
- Use Deepgram API (MANDATORY)
- Must support BOTH English and Urdu
- Must automatically detect language

Step 2: Language Handling
- Detect whether input is English or Urdu
- Apply correct formatting rules

Step 3: Urdu → Roman Urdu (ONLY if Urdu input)
- Use GPT-4o-mini

Step 4: Retrieval-Augmented Generation (RAG)
- Use ChromaDB vector database
- Retrieve top relevant chunks
- Generate response using GPT-4o-mini
- Response MUST be:
  - Calm
  - Complete
  - Context-based ONLY

Strict rule:
→ If information is not found:
   "Mujhe is bare mein kafi maloomat nahi hai"

Step 5: Text-to-Speech (TTS)
- Use ElevenLabs API (MANDATORY)
- Do NOT use gTTS
- Voice must be:
  - Female
  - Natural sounding
- Must support BOTH:
  - English
  - Urdu

--------------------------------------------------
