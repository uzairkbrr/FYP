CURRENT IMPLEMENTATION DETAILS

STT:
- Provider: Deepgram (nova-2 model)
- Language detection: automatic (English vs Urdu)
- API: REST via httpx (backend/stt.py)

Transliteration:
- Urdu script → Roman Urdu: GPT-4o-mini (backend/transliterate.py)
- Roman Urdu → Urdu script: GPT-4o-mini (for TTS input)

RAG:
- Vector store: ChromaDB (collection "sample", 247 docs)
- Embeddings: OpenAI text-embedding-ada-002
- Retrieval: top-15 similarity search
- Response generation: GPT-4o-mini
- Bilingual output: roman_urdu + arabic_urdu (for Urdu)
- English output: plain English text

TTS:
- Provider: ElevenLabs
- Voice: Rachel (21m00Tcm4TlvDq8ikWAM) - female, calm
- Model: eleven_multilingual_v2 (English + Urdu)
- Delivery: base64 data URL in JSON response

Frontend:
- React 19 + Vite 7 + Tailwind CSS 4
- useVoiceRecorder hook (shared by VoiceCard + ChatWidget)
- MediaRecorder API for audio capture
- 30-second recording limit with auto-stop

ARCHITECTURE

Backend (backend/ package):
  backend/__init__.py      - Package marker
  backend/config.py        - Centralized config, env loading
  backend/stt.py           - Deepgram STT via httpx
  backend/transliterate.py - Urdu/Roman conversion via GPT
  backend/rag.py           - ChromaDB retrieval + GPT response
  backend/tts.py           - ElevenLabs TTS via httpx
  backend/main.py          - FastAPI app, /voice-query endpoint

Frontend (new/modified files):
  src/hooks/useVoiceRecorder.js   - Shared recording + API hook
  src/components/VoiceCard.jsx    - Updated: connected to backend
  src/components/ChatWidget.jsx   - NEW: floating chat widget
  src/pages/Home.jsx              - Updated: includes ChatWidget

Pipeline flow:
  Audio → Deepgram STT → Language Detection →
  [If Urdu: Urdu→Roman] → ChromaDB RAG → GPT Response →
  [If Urdu: Roman→Urdu for TTS] → ElevenLabs TTS → Audio

HOW TO RUN

Backend:
  cd FYP2_P1
  python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload

Frontend:
  cd FYP2_P1/frontend
  npm run dev

Open: http://localhost:5173
